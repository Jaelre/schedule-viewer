Query Parameters

Field Description
token

String - Token representing the site to access data for, or a comma delimited list of tokens for many sites.
ex) .../schedule?token=aaaaaaaa-aaaa-aaaa-aaaa-aaaaaaaaaaaa
ex) .../schedule?token=aaaaaaaa-aaaa-aaaa-aaaa-aaaaaaaaaaaa,bbbbbbbb-bbbb-bbbb-bbbb-bbbbbbbbbbbb
current

boolean - If true, returns all shifts that are currently occuring (start_time < now && end_time > now).
NOTE: Can not be used with 'startDate' or 'endDate'
startDate

String - YYYY-MM-DD Format, all shifts that start after or on this day will be included.
NOTE: Can not be used with 'current'
endDate

String - YYYY-MM-DD Format, all shifts that start before or on this day will be included.
NOTE: Can not be used with 'current'
scheduleVersion optional

String - (live, working, or draft) The type of shifts to return.
- live: Live Schedule data that physicians have worked, or are scheduled to work.
- draft: Drafted Schedule Data that is subject to change.
- working: Schedule data that is currently being built in the builder.

Default value: live
tags optional

String - Comma seperated list of Tag names (case sensitive). Schedule data with users working a shift who have at least 1 of these tags will be returned.
NOTE: Can not be used with 'userIDs'

Default value: any
shifts optional

String - Comma seperated list of Shift names (case sensitive). Schedule data with Shifts matching these names will be returned.

Default value: any
userIDs optional

String - Comma sperated list of user ids. Schedule data where only these users are working will be returned.
NOTE: Can not be used with 'tags'

Default value: any

HTTP/1.1 200 OK
  {
     "data": [
         {
             "slot_id": 0,                           // unique id for this person working this shift at this time on this day
             "date": "2021-08-02",                   // the date this shift starts
             "is_on_call": false,                    // is this an on-call shift
             "start_time": "2021-08-02 08:00:00",    // what time with respect to the timezone this site is in, does this shift start
             "end_time": "2021-08-02 21:00:00",      // what time with respect to the timezone this site is in, does this shift end
             "is_weekend": false,                    // is this a weekend shift
             "index": 0,                             // multiple people can work a shift, index 0 mean we display this user at above an index 1 and so on...
             "shift": {                              // object containing more details about the shift
                 "id": 0,                            // unique id for this shift
                 "name": "8am-9pm",                  // name for the shift
                 "alias": "FullDay",                 // alias used by physicians at the site for this shift
                 "color": "#a8ffab"                  // the colour we use to represent this shift in tool
             },
             "user": {                               // user object containing more details about the user working this shift.
                 "id": 0,                            // unique id for the user
                 "fname": "FirstName",               // users first name
                 "mname": "MiddleName",              // users middle name
                 "lname": "LastName"                 // users last name
             },
             "site": {                               // object containing details about the site for the token passed
                 "id": 0,                            // unique id for this site
                 "name": "MetricAid API Test Site",  // name of the site
                 "short_name": "MATS",               // short (abbreviated) name for the site
                 "timezone": {                       // site timezone object
                     "id": 267,                      // a unique id for the timezone this site is in
                     "name": "America/Toronto"       // timezone abbreviation, can tell us what UTC offset this site has.
                 }
             }
         },
         {
             .... // Potentially more object as above with different data. Check data.length to know how many records were retrieved.
         }
     ],
     "paging": null,                         // no paging yet...
     "status": {
         "code": "General.Success",          // the status code is General.Success (That means in general it was successful)
         "message": "Success"                // if there was an error this message will get you a better clue of what went wrong
     },
     "executedAt": "2021-08-24T21:25:36.670Z",   // time the request was received
     "timeElapsed": 82,                      // time from request reeived to a response sent
     "error": false                          // no error so its false, always check this on every response retrieved.
 }

Retreive all Users (and optionally their tags) for each token

get 
https://api.metricaid.com/api/v1/public/users
  • Get all users for many tokens with tags 
https://api.metricaid.com/public/users?token={token1},{token2}&includeTags=true

Query Parameters

Field Description
token

String - Token representing the site to access data for, or a comma delimited list of tokens for many sites.
ex) .../users?token=aaaaaaaa-aaaa-aaaa-aaaa-aaaaaaaaaaaa
ex) .../users?token=aaaaaaaa-aaaa-aaaa-aaaa-aaaaaaaaaaaa,bbbbbbbb-bbbb-bbbb-bbbb-bbbbbbbbbbbb
includeTags

Boolean - Add a property 'tags' to each user. Contains tags the user has at each site.
